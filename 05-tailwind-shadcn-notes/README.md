## 작업 05 – TailwindCSS & shadcn/ui 기본 사용법 요약

### 이 폴더에서 할 일

- Tailwind와 shadcn/ui를 \"읽고 따라칠 수 있는\" 정도로만 익힙니다.
- 자주 쓸 유틸리티/컴포넌트 패턴을 메모해 둡니다.

### 당장 5~10분 안에 할 수 있는 작은 단위

1. 브라우저에서 \"tailwind cheat sheet\" 또는 \"tailwind 기본 문법\"을 검색합니다.
2. 예시 코드 중에서 마음에 드는 버튼/레이아웃 예시 1개만 골라 이 README에 붙여 넣습니다.
3. 예시에서 사용한 클래스 이름 중 3개만 골라 아래처럼 의미를 적습니다.
   - `bg-blue-500  // 배경색 파란색 계열`
   - `px-4        // 좌우 패딩 1rem`
   - `rounded-md  // 중간 정도의 border-radius`

### 체크리스트 (Tailwind)

- [ ] 자주 쓸 것 같은 Tailwind 클래스 5~10개를 표로 정리했다.  
       (예: `bg-*`, `text-*`, `px-*`, `py-*`, `rounded-*`, `flex`, `gap-*` 등)
- [ ] `className`에 여러 유틸리티를 조합한 예시를 2~3개 적어 보았다.
- [ ] 색상/폰트/spacing을 Tailwind로 표현할 때 어떤 패턴을 쓸지 감을 잡았다.

### 체크리스트 (shadcn/ui)

- [ ] shadcn/ui 공식 문서에서 설치/세팅 흐름을 단계별로 정리했다.  
       (예: pnpm add, `npx shadcn@latest init`, 컴포넌트 추가 명령 등)
- [ ] `Button`, `Input`, `Card` 샘플 코드 하나씩 가져와서, 주요 props/클래스를 한국어로 주석 달았다.
- [ ] \"shadcn/ui 컴포넌트 = 그냥 복사해 오는 React 컴포넌트\"라는 점을 이해하고,  
       이 프로젝트의 디자인 시스템에 맞게 수정할 수 있다는 것을 메모했다.

### 힌트

- Tailwind 문법을 **외우려고 하지 말고**, 자주 쓰는 패턴 몇 가지만 고정 메뉴처럼 챙긴다고 생각하면 편합니다.
- shadcn/ui는 라이브러리가 아니라 **소스 코드 템플릿**이라는 점이 중요합니다.  
  나중에 After 패키지에서 복사/수정하는 상상을 해 보세요.
