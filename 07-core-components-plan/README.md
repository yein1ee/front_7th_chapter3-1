## 작업 07 – 핵심 공통 컴포넌트 재설계 계획 (Button, Input, Card, Table)

### 이 폴더에서 할 일

- After 패키지에서 만들 핵심 UI 컴포넌트들의 API(Props)와 역할을 설계합니다.
- variant/size 같은 공통 패턴을 정리합니다.

### 당장 5~10분 안에 할 수 있는 작은 단위

1. 이 README 안에 아래 표 틀을 붙여 넣습니다.
2. `Button` 컴포넌트부터, 떠오르는 prop만 채워 봅니다. (정확하지 않아도 괜찮음)

예시 표 틀:

| 컴포넌트 | prop 이름 | 타입 예시      | 설명          |
| -------- | --------- | -------------- | ------------- | ---------------- | --------- |
| Button   | variant   | \"primary\" \\ | \"secondary\" | 버튼 스타일 종류 |
| Button   | size      | \"sm\" \\      | \"md\" \\     | \"lg\"           | 버튼 크기 |

여기까지 작성하면 1라운드 종료입니다.

### 체크리스트

- [ ] `Button`, `Input`, `Select`, `Card`, `Table` 각각에 대해 **필수 props / 선택 props**를 표로 정리했다.
- [ ] variant, size를 공통 인터페이스로 맞출지(예: 모두 `variant`, `size` 사용) 고민해서 메모했다.
- [ ] 컴포넌트가 **UI만 담당**하게 하기 위해, 비즈니스 로직성 props를 최대한 줄이는 방향으로 정리했다.

### 힌트

- 지금은 구현이 아니라 **API 디자인** 단계입니다.  
  \"화면에서 이 컴포넌트를 사용할 때, 어떤 정보만 넘기면 편할까?\" 관점으로 생각하면 수월합니다.
