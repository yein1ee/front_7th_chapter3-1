## 작업 04 – 디자인 시스템 기본 개념 학습 노트

### 이 폴더에서 할 일

- 디자인 시스템, 디자인 토큰, Atomic Design 같은 기본 개념을 얕게 이해합니다.
- 이 프로젝트 과제와 어떻게 연결될지 감을 잡습니다.

### 당장 5~10분 안에 할 수 있는 작은 단위

1. 브라우저에서 \"design system 이란\" 또는 \"디자인 시스템 설명\"으로 검색합니다.
2. 검색 결과 중 **읽기 쉬워 보이는 글/블로그 1개**만 엽니다.
3. 그 글에서 핵심 문장 1~2개만 복사해서 이 README 아래에 붙여 넣습니다.
4. 그 아래에 **내 말로 한 줄 요약**을 적습니다.
   - 예: \"디자인 시스템 = 여러 화면에서 재사용할 수 있는 UI 규칙/부품 모음이라고 이해했다.\"

이 정도만 해도 충분한 1라운드입니다.

### 체크리스트

- [ ] 디자인 시스템에 대한 설명을 한 문단(3~4줄) 정도로 \"내 언어\"로 요약했다.
- [ ] \"Design Token\"이 무엇인지 찾아보고, 예시(색상 토큰 등)를 적어 보았다.
- [ ] Atomic Design(Atoms/Molecules/Organisms)을 간단히 이해하고, 현재 프로젝트의 폴더 구조와 연결해서 설명해 보았다.
- [ ] PR 템플릿의 목표(디자인 토큰, dark mode, 일관된 컴포넌트 API)와 이 개념들이 어떻게 연결될지 2~3줄로 적었다.

### 힌트

- 이 단계는 **이론 100% 이해**가 목표가 아닙니다.  
  \"대충 이런 말이구나\" 정도만 잡고, 나중에 코드를 만들면서 몸으로 이해해도 됩니다.
