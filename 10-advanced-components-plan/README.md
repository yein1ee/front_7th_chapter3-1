## 작업 10 – 테이블/모달/알림 컴포넌트 API 정리

### 이 폴더에서 할 일

- `Table`, `Modal`, `Alert`를 디자인 시스템 관점에서 어떻게 추상화할지 정리합니다.
- 공통적으로 쓸 수 있는 props와 책임을 정의합니다.

### 당장 5~10분 안에 할 수 있는 작은 단위

1. `packages/before/src/components/organisms/Table.tsx` (또는 After의 Table) 파일을 엽니다.
2. 컴포넌트 선언부의 props 타입/인터페이스만 눈으로 보고, **prop 이름 3~5개만** 이 README에 적습니다.
   - 예: `columns`, `data`, `striped`, `hover`, `onEdit`
3. 각 prop 옆에 한 줄 설명을 추가합니다.

### 체크리스트 (Table)

- [ ] 현재 Table이 받는 props를 표로 정리했다 (이름/타입/설명).
- [ ] \"열 정의\"(columns) 구조를 일반화해, 다양한 도메인에서 재사용할 수 있는 형태로 설계안을 적었다.
- [ ] 행 액션 버튼(onEdit, onDelete, onPublish 등)을 어떻게 확장/커스터마이즈할지 아이디어를 적었다.

### 체크리스트 (Modal & Alert)

- [ ] `Modal`에서 공통으로 쓸 props를 정리했다. (예: `isOpen`, `title`, `size`, `onClose`, `footer` 등)
- [ ] `Alert`의 variant(성공/실패/정보 등), 아이콘, 닫기 버튼 유무를 props로 통일하는 설계를 적었다.
- [ ] 이 세 컴포넌트가 **비즈니스 로직 없이 UI 역할만** 하도록, 어디까지를 책임으로 둘지 문장으로 정리했다.
