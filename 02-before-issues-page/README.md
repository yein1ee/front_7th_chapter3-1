## 작업 02 – Before UI 문제점 관찰 (ManagementPage 중심)

### 이 폴더에서 할 일
- `ManagementPage.tsx`를 중심으로 현재 UI 구조와 스타일의 문제를 눈으로 확인합니다.
- **어디가 복잡한지, 왜 개선해야 하는지**를 말로 적어 봅니다.

### 당장 5~10분 안에 할 수 있는 작은 단위
1. `packages/before/src/pages/ManagementPage.tsx` 파일만 엽니다.
2. 파일 맨 위~아래를 빠르게 스크롤하면서, **`style={{ ... }}` 객체**가 보이는 부분에만 주목합니다.
3. 인라인 스타일이 있는 부분 2~3군데를 골라, 이 README에 복붙합니다.
4. 각 예시 아래에 한 줄씩만 느낌을 적습니다.  
   - 예: \"색상이 숫자로 박혀 있어서 바꾸기 힘들 것 같다.\"

여기까지가 1라운드입니다. 이후에 나머지 로직/구조를 천천히 봅니다.

### 체크리스트
- [ ] 인라인 스타일 예시 2~3개를 이 문서에 붙여 넣었다.
- [ ] 어떤 state들이 있는지 (`useState` 부분)만 훑어보고, 상태 이름을 목록으로 적었다.
- [ ] API 호출/데이터 변경과 관련된 함수 이름들만 골라서 적었다.  
      (예: `loadData`, `handleCreate`, `handleUpdate`, `handleDelete`, `handleStatusAction`)
- [ ] \"이 많은 기능이 한 컴포넌트에 다 몰려 있을 때 생기는 문제\"를 3~5줄로 정리했다.

### 힌트
- **정답 찾기**가 아니라, \"지금 구조가 왜 읽기/수정하기 힘든지\"를 스스로 느껴보는 단계입니다.
- 다 이해가 안 되어도 괜찮습니다. **어디가 불편해 보이는지**만 적어도 충분합니다.
