## 작업 03 – Before 공통 컴포넌트 스타일 문제점 정리

### 이 폴더에서 할 일

- `components.css`와 atoms/molecules/organisms 컴포넌트를 훑어보며, 스타일/설계의 한계를 정리합니다.
- 나중에 \"디자인 토큰 + Tailwind\"로 옮길 때 개선 포인트를 미리 뽑아 둡니다.

### 당장 5~10분 안에 할 수 있는 작은 단위

1. `packages/before/src/styles/components.css` 파일만 엽니다.
2. 스크롤을 내려가면서 **`.btn-`, `.card-`, `.badge-`** 로 시작하는 클래스 이름만 눈으로 스캔합니다.
3. 마음에 드는(혹은 마음에 안 드는) 클래스 이름 3개만 골라서 이 README에 적습니다.
4. 각 클래스 옆에 한 줄 코멘트를 답니다.
   - 예: `.btn-primary  // 파란색이 고정이라, 다크 모드나 테마 변경이 어려워 보임`

여기까지만 해도 1라운드는 완료입니다.

### 체크리스트

- [ ] `components.css` 안의 주요 섹션(버튼, 폼, 카드, 알림, 테이블, 모달 등)을 대략 파악했다.
- [ ] 버튼 관련 클래스(`.btn-*`) 이름과 역할을 3~5개 정도 표로 적었다.
- [ ] 카드/배지/알림/테이블/모달 등에서 **색상, 폰트, 간격**이 하드코딩된 예시를 3개 이상 캡처했다.
- [ ] atoms/molecules/organisms에서 대표 컴포넌트 1~2개씩 골라,  
       - 사용 중인 CSS 클래스  
       - 주요 props  
       - 재사용성이 떨어져 보이는 이유  
       를 간단히 bullet로 정리했다.

### 힌트

- \"지금 구조가 최악이다\"를 증명하려는 게 아니라, **나중에 디자인 시스템으로 옮길 때 어떤 걸 바꾸면 좋을지** 메모하는 단계입니다.
- 모든 컴포넌트를 다 보려고 하면 힘들 수 있으니, **각 레벨에서 1~2개만 찝어서 보는 것**을 목표로 두세요.
