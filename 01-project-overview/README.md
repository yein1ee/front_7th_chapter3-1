## 작업 01 – 과제 이해 & 전체 구조 파악

### 이 폴더에서 할 일
- PR 템플릿과 `before` 패키지 구조를 읽고, 과제의 전체 목표를 이해합니다.
- 내가 이해한 목표를 짧게 한국어로 요약해 둡니다.

### 당장 5~10분 안에 할 수 있는 작은 단위
1. `front_7th_chapter3-1/.github/pull_request_template.md` 파일을 IDE에서 엽니다.
2. `## 과제 목표`, `## 필수 과제`, `## 심화 과제` 제목만 먼저 훑어봅니다.
3. 눈에 띄는 키워드 3개만 적어 봅니다.  
   - 예: `디자인 토큰`, `Tailwind`, `shadcn/ui`
4. `packages/before/src` 폴더를 펼쳐서, `components`, `pages`, `services`, `styles` 폴더가 있는지만 확인합니다.

여기까지만 해도 충분한 1회차 작업입니다. 너무 깊게 이해하려고 하지 말고, **“대략 이런 느낌이구나”** 정도만 잡는 것이 목표입니다.

### 체크리스트
- [ ] PR 템플릿에서 **과제 목표** 부분을 3줄 이내로 요약했다.
- [ ] PR 템플릿의 **필수 과제 체크리스트**를 마크다운으로 재작성했다.  
      (예: `- [ ] TailwindCSS 설정 및 디자인 토큰 정의` 처럼)
- [ ] `packages/before/src` 폴더 구조를 트리 형태로 적었다.
- [ ] \"디자인 시스템\", \"컴포넌트 아키텍처\"가 무엇인지 **내 말로** 각각 3~5줄로 설명해 보았다.

### 힌트
- 이 단계에서는 코드를 수정하지 않아도 됩니다. **읽기 + 기록**만 해도 큰 진전입니다.
- 이해가 잘 안 되는 개념은, 정확한 정의보다 **대충 어떤 문제를 해결하는지**에 집중해서 적어 두면 좋습니다.
