## 작업 11 – Storybook 없이 컴포넌트 카탈로그 작성 계획

### 이 폴더에서 할 일

- Storybook을 쓰지 않고, 프로젝트 안에 간단한 \"컴포넌트 갤러리\" 페이지를 만드는 계획을 세웁니다.
- 각 컴포넌트의 대표 상태/variant를 어떻게 보여줄지 정리합니다.

### 당장 5~10분 안에 할 수 있는 작은 단위

1. 이 README에 다음과 같은 단순 목록을 적습니다.
   - `Button 갤러리`
   - `Input 갤러리`
   - `Card 갤러리`
   - `Table 갤러리`
2. 각 항목 옆에, 보고 싶은 예시 1개만 적습니다.
   - 예: `Button 갤러리 - 기본/primary/secondary/disabled 버튼 4개 한 줄에 보기`

### 체크리스트

- [ ] After 패키지에 `ComponentsGallery` 또는 `PlaygroundPage` 같은 페이지를 하나 만든다고 가정하고, 라우팅/파일 이름을 메모했다.
- [ ] Button, Input, Select, Card, Table, Alert, Modal 각각에 대해 **최소 2가지 상태**(기본/비활성/에러 등)를 어떻게 보여줄지 정리했다.
- [ ] 각 컴포넌트 섹션에 어떤 텍스트 정보를 같이 표시할지(예: 사용 예시 코드, props 설명)를 아이디어로 적었다.

### 힌트

- 이 페이지는 개발자용 **쇼룸**이라고 생각하면 됩니다.  
  실제 서비스 사용자는 안 보더라도, 나와 동료가 컴포넌트를 빠르게 확인할 수 있는 공간입니다.
